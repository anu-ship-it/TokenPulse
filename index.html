Token Tips

Use less. Get more
from every session.

Practical tips to stretch your context window and avoid hitting limits mid-conversation.

01
Start a new chat when the topic changes
Every message in a conversation adds to the context. When you shift topics, start fresh — don't drag old context into a new task.
02
Use Projects for recurring work
Put stable instructions and files in a Claude Project instead of pasting them every session. Claude remembers so you don't have to repeat.
03
Ask for compact answers first
Tell Claude to give the shortest useful answer first, then expand only if needed. Saves tokens on both sides of the conversation.
Give me the shortest useful answer first. Ask if I want more detail.
04
Summarize before a long session
If a conversation is getting long, ask Claude to summarize the key decisions so far. Then start a new chat with just that summary.
Summarize the key decisions and conclusions from this conversation in under 200 words.
05
Use faster models for simple tasks
Don't use Claude Opus for formatting a list or rewriting a short paragraph. Save your premium model quota for analysis and deep work.
06
Watch your 5-hour session limit
Claude's 5-hour rolling window resets independently of the 7-day limit. TokenPulse shows both — check your session usage before a heavy session.
01
Use Custom Instructions for stable context
Put your tone, format preferences, and recurring context in Custom Instructions. Stop re-briefing ChatGPT on every new chat.
02
One job per chat
ChatGPT works best when each conversation has a single clear purpose. Mixing research, drafting, and editing in one chat burns context fast.
03
Skip Thinking mode for simple tasks
Thinking models use more tokens and quota. Use them for hard architecture decisions or complex debugging — not for rewriting emails.
Give me a direct answer. No need for deep reasoning on this one.
04
Use Projects for recurring clients or topics
ChatGPT Projects keep context scoped. One project per client, one per research area. Keeps conversations focused and context lean.
05
Split research from writing
Do your research in one chat, then start a fresh chat for the final output. Don't carry the full research thread into the writing session.
06
Watch TokenPulse's bar during coding sessions
Code and logs consume tokens fast. When the bar hits 70%, start wrapping up or summarize the session state before continuing in a new chat.
